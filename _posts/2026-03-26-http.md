## Köprü Metni Aktarım Protokolü (HTTP) Nedir?

Köprü Metni Aktarım Protokolü (HTTP), web'de istemci ve sunucu arasında iletişim sağlayan bir ağ protokolüdür.

## HTTP Protokolünün Özellikleri

* OSI modeline göre, HTTP protokolü 7.katmanda bulunur.
* Taşıma katmanında TCP protokolünü kullanır.
* İstemci-sunucu (Client-Server) mimarisine sahiptir.
* HTTP protokolü bağlantısız bir protokoldür, OSI modeline göre uygulama katmanı düzeyinde bağlantı kurulumuna gerek yoktur.

## Uniform Resource Locator (URL)

Tekdüzen Kaynak Konum Bulucu (URL), bir web adresine ulaşmak için kullanılan bir adrestir ve yapısında hedef web sunucusu hakkında bazı bilgiler içerir.

![örnek-url](assets/http/url.png)

## HTTP Protokolü Çalışma Mantığı

HTTP protokolü bir istemci-sunucu mimarisine sahip olduğundan ve bağlantısız bir protokol olduğundan, çalışma mantığı oldukça basittir. HTTP istemcisi HTTP sunucusuna bir istek gönderir ve isteği alan HTTP sunucusu HTTP istemcisine bir yanıt döndürür.

![http-çalışma-mantığı](assets/http/http-calisma-mantigi.png)

## HTTP Mesajları

HTTP protokolünün 2 temel mesaj türü vardır:

![http-mesaj-türleri](assets/http/http-mesaj-turleri.png)

**HTTP İsteği**

HTTP isteği, istemci tarafından HTTP sunucusuna gönderilen mesajdır. HTTP isteğinde 4 bölüm olabilir:

* Talep satırı
* HTTP istek başlıkları
* Boş bir çizgi
* İstek mesajı gövdesi

![http-istek](assets/http/http-istegi.png)

**HTTP Yanıtı**

HTTP yanıtı, istemci tarafından gönderilen HTTP isteğine yanıt olarak HTTP sunucusu tarafından gönderilen mesajdır. HTTP yanıtında 4 bölüm olabilir:

* Durum çizgisi
* HTTP yanıt başlıkları
* Boş bir çizgi
* Yanıt mesajı gövdesi

![http-yanit](assets/http/http-yanit.png)

## HTTP Yöntemleri

HTTP yöntemleri, sunucuda çeşitli işlemleri gerçekleştirmek için tanımlanan protokole özgü yöntemlerdir. Birçok HTTP yöntemi arasında birkaçı aşağıda açıklanmıştır:

1) **GET Method**

HTTP istemcisi, sunucudan GET yöntemiyle gönderilecek belirli bir web sayfası talep eder.

2) **POST Method**

POST yöntemi, URL tarafından verilen adrese atanan verilerin sunucu tarafından kabul edilmesini sağlar. Tüm verileri sunucuya iletir.

3) **OPTIONS Method**

OPTIONS yöntemi, sunucu tarafından desteklenen HTTP yöntemlerinin alınmasına izin verir.

HTTP yöntemlerinden bazıları yukarıda verilmiştir. HTTP protokolünün yöntemlerinin listesine ve ayrıntılarına aşağıdaki adresten erişebilirsiniz:
https://en.wikipedia.org/wiki/Hypertext_Transfer_Protocol#Request_methods 

## HTTP İstek Başlıkları

Bir HTTP isteğinin en temel kısımlarından biri istek başlıklarıdır. Bazı istek başlıkları ve açıklamaları aşağıdaki gibidir:

1) **Host**

Sunucunun alan adını içeren başlıktır. Bazı durumlarda, alan adı ve port numarası birlikte dahil edilebilir.

2) **Accept**

İstemci tarafından kabul edilen veri türlerini sunucuya bildiren başlıktır.

3) **Accept-Language**

İstemcinin kabul ettiği dilleri sunucusuna bildiren başlıktır.

4) **Accept-Encoding**

İstemci tarafından desteklenen kodlama algoritmalarını sunucuya bildiren başlıktır. Genel olarak, bu bölüm desteklenen sıkıştırma algoritmalarını içerir.

5) **Connection**

İşlemin sonunda ağ bağlantısının açık olup olmayacağını sunucuya bildiren başlıktır.

6) **Referer**

Talebin yönlendirildiği adresi içerir. Mevcut web sitesindeki diğer adreslerin bağlantılarına tıklandığında, bu başlık eklenebilir ve bir istek gönderilebilir.

7) **User-Agent**

İstemcinin işletim sistemi bilgileri ve tarayıcı bilgileri sunucuya iletilir.

8) **Cache-Control**

Önbellekleme mekanizmasının kontrolü hakkında bilgi içerir.

9) **Authorization**

HTTP kimlik doğrulaması için gerekli kimlik bilgilerini içeren başlıktır.

10) **Cookie**

Sunucu tarafından ayarlanan çerez bilgilerini içeren başlıktır.

HTTP protokolünün istek başlıklarının listesine ve ayrıntılarına aşağıdaki adresten erişebilirsiniz:
https://en.wikipedia.org/wiki/List_of_HTTP_header_fields

## HTTP Yanıt Başlıkları

HTTP isteğine yanıt olarak gönderilen HTTP yanıtındaki bölümlere yanıt başlıkları denir. HTTP yanıtına dahil edilen bu başlıklardan bazıları aşağıdaki gibidir:

1) **Date**

HTTP isteğine yanıt olarak gönderilen HTTP yanıtının tarih ve saatini gösteren başlıktır.

2) **Server**

Sunucu adını ve bazen sürüm bilgilerini içeren başlıktır.

3) **Last-Modified**

Sunucudaki kaynakta yapılan son değişikliğin tarihini gösteren başlıktır.

4) **Content-Encoding**

Sunucu tarafından gönderilen içeriğin hangi kodlama algoritmalarını uyguladığını söyleyen başlıktır.

5) **Content-Length**

Sunucu tarafından istemciye gönderilen yanıttaki gövde parçasının boyutunun bildirildiği başlıktır.

6) **Connection**

İşlemin sonunda ağ bağlantısının açık olup olmayacağını istemciye bildiren başlıktır.

7) **Content-Type**

Müşteriye gönderilen içerik türünün bildirildiği başlıktır.

HTTP protokolünün yanıt başlıklarının listesini ve ayrıntılarını aşağıdaki adreste bulabilirsiniz:https://en.wikipedia.org/wiki/List_of_HTTP_header_fields

## HTTP Güvenlik Başlıkları

HTTP protokolü, güvenlik sağlamak için güvenlik başlıkları kullanır. Bu başlıklardan bazıları aşağıdaki gibidir:

* Sıkı-Taşıma-Güvenlik
* İçerik-Güvenlik-Politika
* X-Frame-Seçenekleri

**NOT**: HTTP protokolü güvenlik başlıklarını kullanarak güvenlik sağlamaya çalışsa da, trafiğin şifrelenmiş olarak iletilmesi özellikle önemlidir. Bunlar, trafiğin şifreli iletimini sağlayan SSL/TLS protokolleridir. SSL/TLS protokollerini kullanan HTTP protokolünün özel adı HTTPS'dir.
